# LIFE NETWORK Pre-Login Screen Design Brief For Stitch

## Goal

Design the pre-login entry screen for `LIFE NETWORK`, a premium Web3 wallet app for BNB Smart Chain users.

This should feel like the actual first screen of a native mobile wallet app. It should not feel like a landing page, a dashboard, or a phone mockup inside a webpage.

## Screen Scope

Only design the first pre-login screen.

The screen has three required visual elements:

- `LIFE NETWORK` logo
- Primary CTA: `Create New Wallet`
- Secondary CTA: `Connect Existing Wallet`

Do not add explanatory paragraphs, feature descriptions, network badges, onboarding copy, tutorial text, or marketing copy on this first screen.

## Required Visible Text

The only visible text on the first screen should be:

- `LIFE`
- `NETWORK`
- `Create New Wallet`
- `Connect Existing Wallet`

## Logo Direction

Create a custom `LIFE NETWORK` wordmark.

The logo should feel premium, futuristic, metallic, and automotive-tech inspired. It can reference the feeling of a sleek connected premium wordmark such as the KIA logo, but it must be an original LIFE NETWORK design and must not copy the KIA logo.

Logo requirements:

- Custom original wordmark for `LIFE`
- `NETWORK` as a smaller supporting label below
- Metallic silver, chrome white, or polished white treatment
- Subtle LIFE lime accent
- Sleek, minimal, high-end, futuristic
- Clearly readable as `LIFE`

Avoid:

- Generic app icon above the logo
- Cartoon icons
- Overly large typography
- Cheap neon effects
- Exact KIA logo imitation
- Abstract marks that make `LIFE` unreadable

## Visual Direction

Overall style:

- Deep dark theme
- Black and deep navy base
- Premium Web3 financial app tone
- Refined glassmorphism
- Native app feel

Color direction:

- Background: near-black, deep navy, blue-black
- Primary accent: LIFE lime
- Secondary accent: subtle cyan only if needed
- Text: high contrast white or cool gray
- Buttons: compact, tactile, premium

Glassmorphism direction:

- Use translucent glass panels sparingly
- The logo may sit inside a refined glass tile or subtle glass surface
- Buttons may use soft translucent borders and depth
- Use blur, inner highlights, and restrained glow
- Avoid excessive floating cards or dashboard-like composition

## Layout

Mobile-first native app layout.

Recommended structure:

- Center area: LIFE NETWORK logo
- Bottom area: two stacked CTA buttons
- Generous empty space
- Actions positioned near the lower portion like a native wallet app
- Compact premium buttons, not oversized accessibility-style buttons
- No fake device frame
- No browser mockup
- No fake status bar

The design should work well at a 390px mobile width.

## Button Requirements

Primary button:

- Text: `Create New Wallet`
- LIFE lime fill or strong LIFE lime accent
- Compact native-app CTA height
- Right arrow icon is acceptable

Secondary button:

- Text: `Connect Existing Wallet`
- Dark translucent glass button
- Subtle border
- Right arrow icon is acceptable

Do not include button subtitles such as:

- `Continue with Google and set a password`
- `Enter mnemonic and set a password`

## Interaction Context

These interactions are not part of this first-screen design, but the layout should visually support two entry paths:

- `Create New Wallet`: later opens a bottom sheet with Google login
- `Connect Existing Wallet`: later opens a mnemonic import flow

The first screen itself should remain minimal.

## Do Not Include

Do not show:

- `Wallet access`
- `지갑을 시작하세요`
- Long descriptions
- Network badges
- Fingerprint or biometric elements
- WalletConnect options
- MetaMask options
- Trust Wallet options
- External wallet logos
- Phone mockup frame
- Fake mobile status bar
- Marketing sections
- Feature cards

## Desired Feeling

The result should feel:

- Premium
- Minimal
- Native
- Secure
- Web3
- Deep, dark, and polished
- Designed for global premium users

The result should not feel:

- Like a landing page
- Like a presentation mockup
- Like a dashboard
- Like an app for elderly users
- Like a generic crypto template
- Like a phone screen displayed inside a web page
